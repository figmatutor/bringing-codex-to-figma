# capture-views.mjs Specification

Auto-generated configuration file for batch screen capture.

## Overview

`capture-views.mjs` is a configuration file auto-generated in the project root that `capture.mjs` dynamically imports at runtime.

## Required Exports

```js
export const VIEW_NAMES = { ... };  // viewKey → display name
export const VIEWS = { ... };       // viewKey → navigation function or null
```

---

## VIEW_NAMES

**Role:** Specifies the Figma frame name for each view.

**Format:**
```js
export const VIEW_NAMES = {
  'view-key': 'Human Readable Name',
};
```

**Usage in capture.mjs:**
```js
const friendlyName = VIEW_NAMES[viewName] || viewName;
const endpointWithName = `${figmaEndpoint}?frameName=${encodeURIComponent(friendlyName)}`;
```

- Falls back to `viewName` if not found in `VIEW_NAMES`
- `label-*` views are auto-converted by capture.mjs (`label-notebooks` → `Notebooks`)

---

## VIEWS

**Role:** Defines Playwright navigation logic for each view.

**Format:**
```js
export const VIEWS = {
  'view-key': null,                        // Default state (no navigation needed)
  'view-key': async (page) => { ... },     // Custom navigation
};
```

**Usage in capture.mjs:**
```js
const navigate = VIEWS[viewName];
if (navigate) {
  await navigate(page);
}
```

**Navigation function signature:**
```ts
type NavigateFn = (page: PlaywrightPage) => Promise<void>;
```

**Notes:**
- `label-*` views should NOT be defined in VIEWS (capture.mjs handles them automatically)
- Executing with a viewName not in VIEWS throws an error (except `label-*` prefixed views)

---

## Example: URL-Routed App

For Next.js, React Router, Vue Router, etc.

```js
/**
 * capture-views.mjs — Auto-generated by bringing-screens-to-figma
 */

export const VIEW_NAMES = {
  'home': 'Home',
  'dashboard': 'Dashboard',
  'settings': 'Settings',
};

export const VIEWS = {
  'home': null,       // Default page — no navigation needed
  'dashboard': null,  // URL-routed — Playwright navigates by URL
  'settings': null,
};
```

---

## Example: State-Driven SPA

For JupyterNotebook-style apps with UI state switching.

```js
/**
 * capture-views.mjs — Auto-generated by bringing-screens-to-figma
 */

export const VIEW_NAMES = {
  'notebook-list': 'Notebook List',
  'simple-editor': 'Simple Editor',
  'pipeline-view': 'Pipeline View',
};

export const VIEWS = {
  // Default state — no navigation needed
  'notebook-list': null,

  // Open a notebook
  'simple-editor': async (page) => {
    await page.locator('[data-testid="notebook-row"]:first-child').click();
    await page.waitForTimeout(800);
  },

  // Switch to pipeline view
  'pipeline-view': async (page) => {
    await page.locator('[data-testid="pipeline-button"]').click();
    await page.waitForTimeout(800);
  },
};
```

## Auto-Generation Process

1. **Skill analyzes project** → discovers routes/states
2. Assistant asks user directly in chat to confirm the detected view list
3. **Generate capture-views.mjs:**
   - URL-routed: all VIEWS values are `null`
   - State-driven: includes Playwright navigation logic
4. **Add to .gitignore** (if not already present)

---

## capture.mjs Execution Flow

```text
1. Load capture-views.mjs (dynamic import from CWD)
2. Parse --batch arguments
3. Validate viewName:
   - label-* → skip validation (skill auto-generates)
   - Others → verify exists in VIEWS
4. Phase 1 (Parallel Preparation):
   - label-* → createSimpleLabelHTML()
   - Others → navigate to localhost + execute VIEWS[viewName] with page
5. Phase 2 (Sequential Capture):
   - Execute window.figma.captureForDesign()
   - Update capture-map.json
```
